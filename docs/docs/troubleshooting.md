# Troubleshooting

If you're having trouble with MeetDesk, check out this troubleshooting guide for common issues.

## Network/Server

:::tip
MeetDesk will start it's own server on the local network when started. The score entry pages, control panel, and score displays use the network to communicate wirelessly. Thus, it's best to use MeetDesk with a *private* network instead of a public one (such as a meet venue's public Wi-Fi). The network does **not** need to have Internet access!
:::

:::tip
If you have an old router lying around, you could use it to easily start up a separate scoring network for MeetDesk to run on.
:::

## Scoring

:::tip
Have your judges write scores on scorecards after judging, so that if a score gets dropped or not entered, the meet director can enter it using the control panel later.
